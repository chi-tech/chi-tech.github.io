3 \section devman2_sec0 Global variables available
4 
5 The major global variables are defined in "chi_tech_main.h"
6 
7 They are:
8  - ChiConsole chi_console The link to the lua scripting engine.
9  - ChiMath chi_math_handler A handler of math related entities
10  - ChiPhysics chi_physics_handler A handler of physics related items
11  - ChiMPI chi_mpi A handler for parallel related items.
12  - ChiLog chi_log A handler for parallel logging events and verbosity.
13 
14 There are also a number fo secondary global variables that assist developers
15 with coding. They are:
16  - ChiTimer chi_program_timer Holds a timer which initiates at program
17  start.
18  \code
19  double time_in_ms = chi_program_timer.GetTime();
20  \endcode
21  - std::string chi_input_file_name Holds the input file name if supplied.
22  - bool chi_termination_posted A flag used during interactive mode.
23  - bool chi_sim_option_interactive A flag indicating whether the code is
24  run in interactive mode.
25  - std::vector<chi_mesh::MeshHandler*> chi_meshhandler_stack and
26  int chi_current_mesh_handler. Holds various mesh handlers. When not using
27  the lua code to create a chi_mesh::MeshHandler be sure to set it to current
28  in order to ensure that the method chi_mesh::GetCurrentHandler() works:
29  \code
30  chi_meshhandler_stack.push_back(new chi_mesh::MeshHandler);
31  int index = chi_meshhandler_stack.size()-1;
32 
33  chi_current_mesh_handler = index;
34  \endcode
35  And using it:
36  \code
37  auto& handler = *chi_mesh::GetCurrentHandler();
38  \endcode

83 \subsection devman2_sec0_5 Connecting to the parallel logging utility
84 
85 Printing information in a parallel environment can be a very involved
86 process. One can't simply use "std::cout <<" on every process otherwise
87 the output to the log will be chaotic. For this reason we employ a common
88  logging utility which return an output string stream using the function
89  call ChiLog::Log.
90 
91 Connecting to chi_log is done as follows
92 \code
93 #include <chi_log.h>
94 
95 extern ChiLog chi_log;
96 \endcode
97 
98 The logger needs to be supplied with an enumeration (LOG_LVL) indicating
99  the type of output. The following enumerations are supported:
100  - LOG_0, Used only for location 0
101  - LOG_0WARNING, Warning only for location 0
102  - LOG_0ERROR, Error only for location 0
103  - LOG_0VERBOSE_0, Default verbosity level
104  - LOG_0VERBOSE_1, Used only if verbosity level equals 1
105  - LOG_0VERBOSE_2, Used only if verbosity level equals 2
106  - LOG_ALL, Verbose level 0 all locations
107  - LOG_ALLWARNING, Warning for any location
108  - LOG_ALLERROR, Error for any location
109  - LOG_ALLVERBOSE_0, Default verbosity level
110  - LOG_ALLVERBOSE_1, Used only if verbosity level equals 1
111  - LOG_ALLVERBOSE_2, Used only if verbosity level equals 2
