39 ## Steady state simulations:
40 
41 The cross sections can be used in simulations solving the steady state
42 Linear Boltzmann Equation of the form:
43 
44 \f[
45 \vec{\Omega}_n \boldsymbol{\cdot} \vec{\nabla} \psi_{ng} +
46 \sigma_{tg} \psi_{ng} =
47 \sum_{\ell=0}^L \sum_{m=-\ell}^{+\ell} \frac{2\ell+1}{4\pi}
48  Y_{\ell m} (\vec{\Omega}_n)
49  \sum_{g'=0}^{G-1} \sigma_{s\ell,g'{\to}g} \phi_{\ell m,g'} +
50  q_{ext,ng} + q_{fission,ng}
51 \f]
52 
53 The two most prominent items required here includes \f$ \sigma_{tg} \f$ and
54  \f$ \sigma_{s\ell,g'{\to}g} \f$. The latter is an entry in a structure we
55  call a generic transfer matrix for moment \f$ \ell \f$ with rows \f$g=0..G-1\f$
56  and columns \f$g'=0..G-1\f$. These two items are often the only items required
57  in a transport simulation.
58 
59  In simulations with fission-sources, cross sections
60  support two formats, the simple combined cross sections without delayed
61  neutrons
62 
63 \f[
64 q_{fission,ng} = \frac{\chi_g}{4\pi}
65  \sum_{g'=0}^{G-1} \nu_{g'} \sigma_{fg'} \phi_{00g'}
66 \f]
67 
68 and those with delayed neutrons, the latter which are currently only used in the
69 KEigen solver.
70 
71 ## KEigen-value related items
72 As stated before, the cross section file supports two formats, the simple combined cross sections without delayed
73 neutrons which are depicted above and those with delayed neutrons shown below
74 
75 \f[
76 q_{fission,ng} = \frac{\chi_g}{4\pi}
77  \sum_{g'=0}^{G-1} \nu_{prompt,g'} \sigma_{fg'} \phi_{00g'}
78  + \sum_{j=0}^{J-1} \frac{\chi_{delayed,jg}}{4\pi} \gamma_j
79  \sum_{g'=0}^{G-1} \nu_{delayed,g'} \sigma_{fg'} \phi_{00g'}
80 \f].
81 
82 Codes may also choose to update precursor concentrations for which their decay
83 constants, \f$ \lambda_j \f$ are required.
84 
85 ## Keyword definitions
86 
87 - NUM_GROUPS num_groups Required. Specifies the number of groups for this
88  cross-section. Symbol \f$ G \f$.
89 - NUM_MOMENTS num_moments Required. The number of transfer matrices to allocate
90  for this cross-section (whether used or not). Typically this number is one
91  greater than the scattering order (i.e., \f$ L+1 \f$)
92 - NUM_PRECURSORS num_precursors Optional. Indicates how many precursors are used
93  in this cross section. Symbol \f$ J \f$
94 - Optional key words per line:
95  - SIGMA_T_BEGIN. Optional. Starts a block that is terminated by a line SIGMA_T_END.
96  Each line in the block processes the first two words as [Group, sigma_t].
97  Populates the sigma_tg field. Symbol \f$ \sigma_{tg} \f$.
98  - SIGMA_F_BEGIN. Optional. Starts a block that is terminated by a line SIGMA_F_END.
99  Each line in the block processes the first two words as [Group, sigma_f].
100  Populates the sigma_fg field. Symbol \f$ \sigma_{fg} \f$.
101  - SIGMA_A_BEGIN. Optional. Starts a block that is terminated by a line SIGMA_A_END.
102  Each line in the block processes the first two words as [Group, sigma_a].
103  Populates the sigma_ag field. Symbol \f$ \sigma_{ag} \f$. If this is
104  not supplied then sigma_a is estimated from the transfer matrix and
105  may erroneously estimate balance.
106  - NU_BEGIN. Optional. Starts a block that is terminated by a line NU_END. Each line in
107  the block processes the first two words as [Group, nu]. Populates the
108  nu field. Upon completing the file processing the nu_sigma_fg field
109  gets populated from the product of nu and sigma_fg. Symbol \f$ \nu_g \f$.
110  - NU_PROMPT_BEGIN. Optional. Starts a block that is terminated by a line
111  NU_PROMPT_END. Each line in the block processes the first two words as
112  [Group, nu]. Populates the nu_prompt field. Upon completing the file
113  processing the nu_p_sigma_fg field gets populated from the product of
114  nu_prompt and sigma_fg.
115  Symbol \f$ \nu_{prompt,g} \f$.
116  - NU_DELAYED_BEGIN. Optional. Starts a block that is terminated by a line
117  NU_DELAYED_END. Each line in the block processes the first two words as
118  [Group, nu]. Populates the nu_delayed field. Upon completing the file
119  processing the nu_d_sigma_fg field gets populated from the product of
120  nu_delayed and sigma_fg.
121  Symbol \f$ \nu_{delayed,g} \f$.
122  - CHI_PROMPT_BEGIN. Optional. Starts a block that is terminated by a line
123  CHI_PROMPT_END. Each line in the block processes the first two words as
124  [Group, chi]. Populates the chi_g field. Symbol \f$ \chi_{g} \f$.
125  - DDT_COEFF_BEGIN. Optional. Starts a block that is terminated by a line
126  DDT_COEFF_END. Each line in the block processes the first two words as
127  [Group, ddt_coeff]. Populates the ddt_coeff field.
128  Symbol \f$ \frac{1}{v_g} \f$.
129 
130  - PRECURSOR_LAMBDA_BEGIN. Optional. Starts a block that is terminated by a line
131  PRECURSOR_LAMBDA_END. Each line in the block processes the first two words as
132  [precursor, lambda]. Populates the lambda field (the precursor decay
133  constant). Symbol \f$ \lambda_j \f$.
134  - PRECURSOR_GAMMA_BEGIN. Optional. Starts a block that is terminated by a line
135  PRECURSOR_GAMMA_END. Each line in the block processes the first two words as
136  [precursor, gamma]. Populates the gamma field (the precursor production
137  fraction per fission). Symbol \f$ \gamma_j \f$.
138 
139  - CHI_DELAYED_BEGIN. Optional. Starts a block that is terminated by a line
140  CHI_DELAYED_END. Each line in the block processes the first word as the
141  group index and the remaining NUM_PRECURSORS words as the the individual
142  precursor's associated delayed spectrum (chi). Populates the chi_d field.
143  Symbol \f$ \chi_{delayed,jg} \f$.
144 
145  - TRANSFER_MOMENTS_BEGIN. Optional. Starts a block that is terminated by a line
146  TRANSFER_MOMENTS_END. Each line in the block processes a line only if it
147  starts with the keyword M_GPRIME_G_VAL which needs to be followed by four
148  values [moment,gprime,g,value]. Populates transfer-matrix for moment m,
149  row g, column gprime, with value. Symbol \f$ \sigma_{s\ell,g'{\to}g} \f$.
150 - Comments can be between individual blocks but only the TRANSFER_MOMENTS block
151  may have comments between the _BEGIN and _END
152 - Comments do not have to start with any specific character since the file
153  format is keyword driven.
154 - Any number that is not convertible to its required type (integer, double)
155  will throw an error to that effect.
156 - All errors will indicate the current file, line number and nature of the error.
