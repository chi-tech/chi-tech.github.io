15 /** Implementation of an abstracted field function object holding
16  * a reference to a contiguous vector of data associated with a
17  * mesh and a spatial discretization method. This
18  * object's design has undergone numerous iterations and is still
19  * evolving. The first iteration involved the structuring of groups
20  * and moments associated with transport simulations which exhibited
21  * significant performance gains when the groups were contiguous. The
22  * mapping for each dof was then dof*num_grps*num_moms + num_groups*moment +
23  * group. After this it became clear that the design can be generalized
24  * even further. The deal.ii library has the concepts of unknown ``components"
25  * on each degree of freedom. It also extends the concept to vector
26  * valued components.
27  *
28  * For ChiTech we will generalize a field function to store sets of components.
29  * Each set (akin to a moment) will have all its components contiguous and
30  * will allow algorithms to to build accordingly.*/
