22 /** Implementation of an abstracted field function object holding
23  * a reference to a contiguous vector of data associated with a
24  * mesh and a spatial discretization method. This
25  * object's design has undergone numerous iterations and is still
26  * evolving. The first iteration involved the structuring of groups
27  * and moments associated with transport simulations which exhibited
28  * significant performance gains when the groups were contiguous. The
29  * mapping for each dof was then dof*num_grps*num_moms + num_groups*moment +
30  * group. After this it became clear that the design can be generalized
31  * even further. The deal.ii library has the concepts of unknown ``components"
32  * on each degree of freedom. It also extends the concept to vector
33  * valued components.
34  *
35  * For ChiTech we will generalize a field function to store sets of components.
36  * Each set (akin to a moment) will have all its components contiguous and
37  * will allow algorithms to to build accordingly.*/
