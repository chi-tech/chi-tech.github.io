1 /***********************************************************************
2  * This header file contains the mcp3008Spi class definition.
3  * Its main purpose is to communicate with the MCP3008 chip using
4  * the userspace spidev facility.
5  * The class contains four variables:
6  * mode -> defines the SPI mode used. In our case it is SPI_MODE_0.
7  * bitsPerWord -> defines the bit width of the data transmitted.
8  * This is normally 8. Experimentation with other values
9  * didn't work for me
10  * speed -> Bus speed or SPI clock frequency. According to
11  * https://projects.drogon.net/understanding-spi-on-the-raspberry-pi/
12  * It can be only 0.5, 1, 2, 4, 8, 16, 32 MHz.
13  * Will use 1MHz for now and test it further.
14  * spifd -> file descriptor for the SPI device
15  *
16  * The class contains two constructors that initialize the above
17  * variables and then open the appropriate spidev device using spiOpen().
18  * The class contains one destructor that automatically closes the spidev
19  * device when object is destroyed by calling spiClose().
20  * The spiWriteRead() function sends the data "data" of length "length"
21  * to the spidevice and at the same time receives data of the same length.
22  * Resulting data is stored in the "data" variable after the function call.
23  * ****************************************************************************/
