242  *
243  * The supplied vector should contain the upper bin boundary for each
244  * bin and will return the bin associated with the bin that brackets
245  * the supplied probability.
246  *
247  * Example:
248  * Suppose we sample bins 0-9. Suppose also that the probalities for each
249  * bin is as follows:
250  * - 0.1 bin 0
251  * - 0.1 bin 1
252  * - 0.5 bin 5
253  * - 0.3 bin 8
254  *
255  * The CDF for this probability distribution will look like this
256  * - bin 0 = 0.1
257  * - bin 1 = 0.2
258  * - bin 2 = 0.2
259  * - bin 3 = 0.2
260  * - bin 4 = 0.2
261  * - bin 5 = 0.7
262  * - bin 6 = 0.7
263  * - bin 7 = 0.7
264  * - bin 8 = 1.0
265  * - bin 9 = 1.0
266  *
267  * Supplying a random number between 0 and 1 should indicate sampling one
268  * of the bins 0,1,5 or 8. The most inefficient way to do this is to

270  * An optimized version of this sampling would be to perform a recursive
271  * block search which starts with a course view of the cdf and then gradually
