Writes a string to the screen in screen co-ordinates. This routine writes quad based fonts to the screen.

Parameters
xScreen x-coordinate where text is to be displayed.
yScreen y-coordinate where text is to be displayed.
displayStringPointer to the string to be displayed.
fontScaleFont size multiplication factor.
fontIndexFont type number to be used.
columnWidthMaximum column size to display text. [Default: 0,no column wrapping]
lineFeedDirection of line feed. [Default: 2,downwards]
cursorPosPosition of the cursor inside the string.

!!Parameter description

  • }fontScale}
    Multiplies the size of the texture mapped quad by this factor. If 1.0, no scaling is applied, otherwise the quad is stretched or shrinked according to the scale. The effects of this scale become more negative the more extreme the scale since rasterization artifacts appear.

  • }fontIndex}
    Index of the font, in the font stack, to be used for writing the text. The following choices are available: The font stack

  • }columWidth}
    Limiting column size after which text will be line fed. If this parameter is zero, no column width line feed is performed, however, normal line feed via the lineFeed parameter is still performed if set. If linefeed is zero, setting this parameter has no effect.

  • }lineFeed}
    Line feed option. The following parameters are allowed: ||Parameter value ||Description |0 |No line feed is performed. Line feed characters are displayed as spaces. |1 |Line feed is upwards. Each line feed character displays the next line above the previous. |2 |Line feed is downwards. Each line feed character displays the next line below the previous.
