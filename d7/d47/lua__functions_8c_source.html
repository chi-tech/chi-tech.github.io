164 /**\defgroup LuaSLDFESQ Simplified LDFESQ
165  * \ingroup LuaQuadrature
166  *
167  * This quadrature is a simplified implementation of the quadrature
168  * defined in the paper by Cheuk Lau and Marvin Adams:
169  * "Discrete Ordinates Quadratures Based on Linear
170  * and Quadratic Discontinuous Finite Elements over
171  * Spherical Quadrilaterals", Nuclear Science and Engineering, 185:1,
172  * pages 36-52, 2017.
173  *
174  * \image html "SLDFESQn40.png" "LDFE Spherical Quadrilaterals at a refinement level of n=40" width=800px
175  *
176  *
177  * # Synopsis
178  * - The initial refinement is generated for the octant where all the
179  * direction cosines are positive. We will call this the first octant.
180  * - A cube of side length \f$a=\frac{1}{\sqrt{3}}\f$ is inscribed within
181  * the unit sphere and the origin planes. This cube has three faces
182  * that project onto the unit sphere in the first octant.
183  * \image html "SLDFESQFig2.png" "The inscribed cube with one of the faces subdivided" width=800px
184  * - Each face of the cube can be described by a local coordinate system,
185  * \f$\tilde{x}\f$, \f$ \tilde{y}\f$. In this reference frame each face
186  * is identically subdivided into an orthogonal grid. The generated
187  * grid-cells are called sub-squares.
188  * - A face and its associated sub-squares, in the local reference frame,
189  * can be rotated and translated
190  * onto the cube faces, for which we will call the reference frame the
191  * \f$xyz\f$-prime reference frame.
192  * - The sub-squares on each face project onto
193  * Spherical Quadrilaterals (SQs) on the unit sphere, a space which we
194  * shall refer to as the \f$xyz\f$ reference frame. Or in the paper
195  * mu-eta-xi.
196  * \image html "SLDFESQFig3.png" width=400px
197  * - Once the SQs are generated we have numerous choices on where to
198  * define quadrature points and associated shape functions. In the LDFE
199  * formulation we need 4 quadrature points and, for simplicity, we restrict
200  * the placement of these points in a certain manner. We form radii, in
201  * the xy-tilde reference, from the sub-square centroids to each corner
202  * of the sub-square (essentially producing sub-sub-squares) and then
203  * chose to place our quadrature points only on these points.
204  * - The placement of the quadrature points can then be projected to
205  * xyz after which the shape function coefficients can be determined.
206  * - The weight of each quadrature point is the spherical surface area
207  * integral of its shape function. This integral is done with a high order
208  * Guass-Legendre quadrature and a Jacobian \f$ \frac{a}{r^3} \f$.
209  * - The original Lau and Adams paper used a Multi-variate secant method to
210  * determine the placement of the quadrature points along the sub-sub-square
211  * radii.
212  *
213  * # Specific notes for the simplified version
214  *
215  * ## Initial sub-division of the inscribed cube face
216  * The orthogonal grid in the xy-tilde reference frame is dependent on a
217  * a diagonal spacing of grid points. The paper by Lau and Adams did not
218  * specify exactly how to generate this spacing but only stated that their
219  * spacing minimizes the max/avg and max/min ratio of projected SQ to
220  * 1.1 and 1.3 respectively. We found that the application of a weighting
221  * function, \f$ \alpha (\cos \beta \frac{\pi}{2} x - \cos \beta \frac{\pi}{2})\f$
222  * produces the optimal spacing. See the code for additional clarification.
223  *
224  * ## Placement of the quadrature points
225  * Instead of using a multi-variate secant method to place the quadrature
226  * points we found that placing the points on the 4 Gauss-Legendre points
227  * for a quadrilateral (\f$[\pm \frac{1}{\sqrt{3}},\pm \frac{1}{\sqrt{3}} ]\f$)still produced 4th order convergence and added a
228  * lot of speed to the algorithm.
229  *
230  * ## Integration of basis functions
231  * The determinant of the Jacobian can be hard to derive. It can be done
232  * in the two ways:
233  * [using a cross-product](../../HTMLimages/SLDFESQ_JacobianA.jpg)
234  * or in [angle-space](../../HTMLimages/SLDFESQ_JacobianB.jpg).
235  *
236  *
237  * ## Local refinement
238  * The data structures employed allows the SQs to easily be refined in certain
239  * regions.
240  *
241  *
242  * */
