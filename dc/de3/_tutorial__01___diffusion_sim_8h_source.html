2 
3  Let us tackle the basic diffusion equation as applied to steady state heat
4  transfer on a 3D orthogonal mesh.\n
5  \f[
6  \nabla k \nabla T = q
7  \f]
8 
9  The purpose of this tutorial is to understand the basics of the
10  input paradigm which is completely different from the traditional
11  input-card based method.
12  ChiTech uses an imbedded Lua console. For a history of Lua please see its
13  wikipedia page
14  <a href="https://en.wikipedia.org/wiki/Lua_(programming_language)">here</a>.
15  This paradigm allows the user to control many aspects of the program flow
16  as well to be more versatile when creating inputs, i.e. reading files, for
17  loops to create multiple entities etc. So let us get started.
18 
19  ## Step 1 - Create your input file, open it in a text editor
20 
21  Go to the directory of your
22  choosing and make an input file of your choosing. The input file does not
23  require a specific .lua or .inp extension although if you use .lua then
24  there are many
25  text editors that will be able to provide syntax highlighting.
26 
27 ## Step 2 - Create a mesh handler
28 
29 \code
30 chiMeshHandlerCreate()
31 \endcode
32 
33 This function call creates a new mesh handler and pushes it to a global
34 stack. It also make the newly created handler the "current" meaning all
35  mesh operations will operate on this handler.
36  The concept of creating, reading or manipulating computational meshes is
37 centered on operating on a specific chi_mesh::MeshHandler object. Physics/math
38  objects can then freely operate on meshes by simply specifying which handler
39  to use.
40 
41 ## Step 3 - Create a 3D mesh
42 
43  For this we will create a simple orthogonal grid of 32x32x32 cells
44  (32,768 total) as shown in Figure 1.
45 
46 
47  \image html "Meshing/CubeMeshTut1.png" "Figure 1 - Mesh spanning from [-1,-1,-1] to [1,1,1]" width=350px
48 
49 The first step of this process is to define an array of nodes starting from -1.0
50 with a distance \f$ ds \f$ between each of them. In lua we can create this array
51 programatically:
52 \code
53 nodes={}
54 N=32
55 ds=2.0/N
56 for i=0,N do
57  nodes[i+1] = i*ds
58 end
59 \endcode
60 
61 Note here that lua arrays cannot use \f$ nodes[0] \f$ since arrays in lua are not
62 zero based. The next step is call chiMeshCreate3DOrthoMesh with the node
63 arrangement along the X-direction, Y-direction and Z-direction, which in our
64  case is the same,
65 
66 \code
67 surf_mesh, region1 = chiMeshCreate3DOrthoMesh(nodes,nodes,nodes)
68 \endcode
69 
70 Some work has been encapsulated behind the scenes here. Firstly, a 2D surface
71 mesh was created and assigned to a region.
72 Secondly a surface-mesher (chi_mesh::SurfaceMesher)
73  and a volume mesher (chi_mesh::VolumeMesher) has been
74 assigned to the mesh handler. The volume mesher is not executed because the
75 macro allows the user to first assign parallel partitioning.
76 
77 ## Step 4 - Executing the mesher
78 The underlying mesher for a 3D mesh is an extruded mesh from 2D. This mesh
79  has some partitioning properties that can be set to allow parallel simulations,
80  however, we will just focus on serial simulations for now.
81  The next step to complete meshing is to execute the volume mesher. Since we
82  are not going to set any partitioning parameters we will just execute it now.
83 
84 \code
85 chiVolumeMesherExecute();
86 \endcode
87 
88 ## Step 5 - Assign material ID's
89 
90 Materials ID's can conveniently be specified using logical volumes. There are
91 many options for logical volumes ranging from primitive parametric surfaces
92  to non-convex user generated surfaces using surface meshes
93  (see chiLogicalVolumeCreate). For this tutorial
94  we will use a rectangular paralellipiped (RPP or brick) as follows.
95 
96 \code
97 material = chiPhysicsAddMaterial("Test Material");
98 
99 vol0 = chiLogicalVolumeCreate(RPP,-1000,1000,-1000,1000,-1000,1000)
100 chiVolumeMesherSetProperty(MATID_FROMLOGICAL,vol0,material)
101 \endcode
102 
103 We first create a material using the CHI_LUA::chiPhysicsAddMaterial function.
104  The handle of this material will essentially be zero but it is not technically
105  required by the chiVolumeMesherSetProperty() function since this function
106  operates on the integer supplied.
107 This material is added to the physics environment and therefore has scope over
108 all mesh handlers and all physics entities. We then create a logical volume
109  using the function CHI_LUA::chiLogicalVolumeCreate with arguments RPP,
110  specifying that we will be using a Rectangular Parallelipiped and then a
111  series of dimensions specifying xmin-xmax-ymin-ymax-zmin-zmax. Every cell
112  centroid within these dimensions will be flagged as being "within" the logical
113  volume.
114 
115 Actually setting the mesh's material id's is a utility facilitated by a
116  volume mesher property and hence we call CHI_LUA::chiVolumeMesherSetProperty
117  with a property index MATID_FROMLOGICAL. Next we provided a handle to the
118  logical volume (vol0) and the desired material id. Logical volumes are very
119  diverse and their uses are discussed elsewhere.
120 
121 The culmination of this step is all done within a physics agnostic framework.
122 The user can even export the mesh for visualization using the function
123  chiRegionExportMeshToObj(). The extruded mesh is shown below:

126 ## Step 5 - Adding material properties
127 
128 Now that the cells have been assigned a material id we need to add
129  properties to the material conducive to a diffusion simulation. For a heat
130  transfer diffusion simulation we will need to know the thermal conductivity
131  "k" and the volumetric source strength "q". Both of these can be simple
132  scalar values.
133 
134 \code
135 chiPhysicsMaterialAddProperty(material,SCALAR_VALUE,"k")
136 chiPhysicsMaterialSetProperty(material,"k",SINGLE_VALUE,1.0)
137 
138 chiPhysicsMaterialAddProperty(material,SCALAR_VALUE,"q")
139 chiPhysicsMaterialSetProperty(material,"q",SINGLE_VALUE,1.0)
140 \endcode
141 
142 In this code we created the material properties using the function
143  CHI_LUA::chiPhysicsMaterialAddProperty which requires a handle to the reference
144  material, the property type (SCALAR_VALUE), and a name for the property.
145  Unlike extrusion layers the property name can be used in further calls to refer
146  to the specific property.
147 
148 Material property values are set using the function
149  CHI_LUA::chiPhysicsMaterialSetProperty which again expects a handle to the
150  reference material, then either a material property id or name (in this case
151  name), then an operation index and value(s). For this case we used an operation
152  index SINGLE_VALUE which is the only operation supported by SCALAR_VALUE. In
153  future the user can specify, as an example, temperature dependent values which
154  will support the operation FROM_TABLE, but that is a topic for a different time.
155 
156 ## Step 6 - Setup the diffusion physics
157 
158 The following sequence of function calls completely define the diffusion solver.
159 
160 \code
161 phys1 = chiDiffusionCreateSolver();
162 chiSolverAddRegion(phys1,region1)
163 chiDiffusionSetProperty(phys1,DISCRETIZATION_METHOD,PWLC);
164 chiDiffusionSetProperty(phys1,RESIDUAL_TOL,1.0e-4)
165 \endcode
166 
167 We first create the diffusion solver with a call to
168  CHI_LUA::chiDiffusionCreateSolver. This creates the solver and pushes it onto
169  the physics handler. The function returns the handle.
170  We then add our mesh region to this solver using the function
171  CHI_LUA::chiSolverAddRegion which expects a handle to the reference solver
172  followed by the handle to the relevant region.
173 
174 Next we can set numerous diffusion solver properties which can comprehensively
175  viewed in its specific documentation (CHI_LUA::chiDiffusionSetProperty).
176 
177 ## Step 7 - Initialize and Solve
178 
179 The final step of this process is to initialize and execute the diffusion solver.
180 
181 \code
182 chiDiffusionInitialize(phys1)
183 chiDiffusionExecute(phys1)
184 \endcode

186 ## Step 8 - Post-processing
187 The execution of a Chi-Tech physics module culminates in the creation of one or
188  more field functions. These field functions should all be populated for use
189  during the initialization phase of the solver. Users can get handles to all
190  the field functions by using the chiGetFieldFunctionList call. Thereafter, the
191  field function can be exported to VTK-format which can be read by Paraview.
192 
193 \code
194 fflist,count = chiGetFieldFunctionList(phys1)
195 chiExportFieldFunctionToVTK(fflist[1],"Tutorial1Output","Temperature")
196 \endcode

200 ## Complete input file
201 
202 Here is the complete input file with comments
203 
204 \code
205 --############################################### Setup mesh
206 chiMeshHandlerCreate()
207 
208 newSurfMesh = chiSurfaceMeshCreate();
209 chiSurfaceMeshImportFromOBJFile(newSurfMesh,
210  "CHI_RESOURCES/TestObjects/SquareMesh2x2Quads.obj")
211 
212 -- Setup Region
213 region1 = chiRegionCreate()
214 chiRegionAddSurfaceBoundary(region1,newSurfMesh);
215 
216 -- Create meshers
217 chiSurfaceMesherCreate(SURFACEMESHER_PREDEFINED);
218 chiVolumeMesherCreate(VOLUMEMESHER_EXTRUDER);
219 
220 -- Setup extrusion layers
221 subdivisions=3
222 chiVolumeMesherSetProperty(EXTRUSION_LAYER,3.0,subdivisions,"My Layer");
223 
224 -- Execute meshing
225 chiSurfaceMesherExecute();
226 chiVolumeMesherExecute();
227 
228 material = chiPhysicsAddMaterial("Test Material");
229 
230 -- Set Material IDs
231 vol0 = chiLogicalVolumeCreate(RPP,-1000,1000,-1000,1000,-1000,1000)
232 chiVolumeMesherSetProperty(MATID_FROMLOGICAL,vol0,material)
233 
234 -- Export visualization
235 chiRegionExportMeshToObj(region1,"Tutorial01Mesh.obj",true)
236 
237 --############################################### Add material properties
238 
239 
240 -- Set material properties
241 chiPhysicsMaterialAddProperty(material,SCALAR_VALUE,"k")
242 chiPhysicsMaterialSetProperty(material,"k",SINGLE_VALUE,1.0)
243 
244 chiPhysicsMaterialAddProperty(material,SCALAR_VALUE,"q")
245 chiPhysicsMaterialSetProperty(material,"q",SINGLE_VALUE,1.0)
246 
247 
248 --############################################### Setup Physics
249 phys1 = chiDiffusionCreateSolver();
250 chiSolverAddRegion(phys1,region1)
251 chiDiffusionSetProperty(phys1,DISCRETIZATION_METHOD,PWLC);
252 chiDiffusionSetProperty(phys1,RESIDUAL_TOL,1.0e-4)
253 
254 -- Initialize and Execute Solver
255 chiDiffusionInitialize(phys1)
256 chiDiffusionExecute(phys1)
257 \endcode
