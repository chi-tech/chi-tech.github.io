25 /**Object for controlling logging.
26  *
27  * ## Part A: Output logs
28  * There are three levels of verbosity in ChiTech: Zero(Default), One and Two.
29  * These can be set on the command line via the switch -v followed by a
30  * space and the number for the verbosity (0,1 or 2).
31  *
32  * \code
33  * ./bin/ChiTech InputFile.lua -v 1
34  * \endcode
35  *
36  * The lua command `chiLogSetVerbosity(int_level)` achieves the same.\n\n
37  *
38  * Printing a log under the auspices of a verbosity level again has
39  * numerous options. Firstly, any log can be a normal log, a warning
40  * or an error. Secondly, a log can be either location 0 or all the locations
41  * in a parallel process environment. The log option enums defined under
42  * LOG_LVL are
43  - LOG_0, Used only for location 0
44  - LOG_0WARNING, Warning only for location 0
45  - LOG_0ERROR, Error only for location 0
46  - LOG_0VERBOSE_0, Default verbosity level
47  - LOG_0VERBOSE_1, Used only if verbosity level equals 1
48  - LOG_0VERBOSE_2, Used only if verbosity level equals 2
49  - LOG_ALL, Verbose level 0 all locations
50  - LOG_ALLWARNING, Warning for any location
51  - LOG_ALLERROR, Error for any location
52  - LOG_ALLVERBOSE_0, Default verbosity level
53  - LOG_ALLVERBOSE_1, Used only if verbosity level equals 1
54  - LOG_ALLVERBOSE_2, Used only if verbosity level equals 2
55 
56  * A log can be made by first connecting code with the logger. This is done
57  * by including the log header and then defining an extern reference to the
58  * global object.
59  *
60  * \code
61  * #include <chi_log.h>
62  * extern ChiLog& chi_log;
63  * \endcode
64  *
65  * A log is then written inside a piece of code as follows:
66  *
67  * \code
68  * void PrintSomethingToLog()
69  * {
70  * chi_log.Log(LOG_0) << "This is printed on location 0 only";
71  * chi_log.Log(LOG_0WARNING) << "This is a warning";
72  * chi_log.Log(LOG_0ERROR) << "This is an error";
73  * }
74  * \endcode
75  *
76 \verbatim
77 [0] This is printed on location 0 only
78 [0] **WARNING** This is a warning
79 [0] **!**ERROR**!** This is an error
80 \endverbatim
81  *
82  * ## Part B: Repeating events log
83  * Suppose a routine or segment of code gets called multiple times. Now suppose
84  * we want to know how many times this routine was called, how long it ran
85  * (total and on-avg), or we want to know at which timestamp it ran every time.
86  * We do this by using a repeating event. A repeating event is identified
87  * by a tag. Therefore the first step is to obtain a unique id for the event using
88  * ChiLog::GetRepeatingEventTag.
89  *
90  * \code
91  * size_t tag = chi_log.GetRepeatingEventTag(std::string("Sweep Timing"))
92  * \endcode
93  *
94  * Events can now be logged using
95  * ChiLog::LogEvent(size_t ev_tag, ChiLog::EventType ev_type). As an example,
96  * consider we want to count the number of occurrences of an event. We trigger
97  * multiple occurences using
98  *
99  * \code
100  * chi_log.LogEvent(tag,ChiLog::EventType::SINGLE_OCCURRENCE);
101  * \endcode
102  *
103  * At the end of the block for which you wanted to log these events we can now
104  * extract the number of occurrences using
105  *
106  * \code
107  * double number_of_occ = chi_log.ProcessEvent(tag,ChiLog::EventOperation::NUMBER_OF_OCCURRENCES);
108  *\endcode
109  *
110  * Below is a complete example:
111  *
112 \code
113 size_t tag = chi_log.GetRepeatingEventTag(std::string());
114 
115 for (int i=0; i<5; ++i)
116  chi_log.LogEvent(tag,ChiLog::EventType::SINGLE_OCCURRENCE);
117 
118 chi_log.Log(LOG_ALL)
119  << chi_log.ProcessEvent(tag, ChiLog::EventOperation::NUMBER_OF_OCCURRENCES);
120 \endcode
121 \verbatim
122 6
123 \endverbatim
124 
125  * Since chi_log is a global object this functionality is really powerful for
126  * use in modules or class objects where multiple methods can contribute to
127  * the same event track. For example an object can have an event tag as a member
128  * and initialize it at construction then all of the objects methods can
129  * contribute to the event.
130  *
131  * ### Supplying event information
132  * In addition to the ChiLog::EventType the user can also supply a reference to
133  * a ChiLog::EventInfo structure. Be cautious with this though because each
134  * event stored a double and a string which can look like a memory leak if
135  * multiple events are pushed up with event information. Developers can supply
136  * either a double or a string or both to an event info constructor to
137  * instantiate an instance. The event arb_value is by default 0.0 and the event
138  * arb_info is by default an empty string. An example is shown below:
139  *
140 \code
141 size_t tag = chi_log.GetRepeatingEventTag(std::string());
142 
143 for (int i=0; i<5; ++i)
144 {
145  auto ev_info = std::make_shared<ChiLog::EventInfo>(i*2.0);
146  chi_log.LogEvent(tag,ChiLog::EventType::SINGLE_OCCURRENCE,ev_info);
147 }
148 
149 chi_log.Log(LOG_ALL)
150  << chi_log.ProcessEvent(tag, ChiLog::EventOperation::AVERAGE_VALUE);
151 \endcode
152 \verbatim
153 4.0
154 \endverbatim
155  *
156  * Event information can also be supplied by string values but then the
157  * average value is meaningless unless the average value is also supplied.
158  * For example:
159  *
160 \code
161 size_t tag = chi_log.GetRepeatingEventTag(std::string());
162 
163 chi_log.LogEvent(tag,
164  ChiLog::EventType::SINGLE_OCCURRENCE,
165  std::make_shared<ChiLog::EventInfo>(std::string("A"),2.0));
166 chi_log.LogEvent(tag,
167  ChiLog::EventType::SINGLE_OCCURRENCE,
168  std::make_shared<ChiLog::EventInfo>(std::string("B")));
169 chi_log.LogEvent(tag,
170  ChiLog::EventType::SINGLE_OCCURRENCE,
171  std::make_shared<ChiLog::EventInfo>(std::string("C"),2.0));
172 
173 chi_log.Log(LOG_ALL)
174  << chi_log.ProcessEvent(tag, ChiLog::EventOperation::AVERAGE_VALUE);
175 \endcode
176 \verbatim
177 1.33333
178 \endverbatim
179  *
180  * To get a string value of the event history developers can use
181  * ChiLog::PrintEventHistory along with the event tag. Just note that it will
182  * automatically be formatted for each location so no need to use chi_log to
183  * print it. Also, each event will be prepended with a program timestamp
184  * in seconds.
185  *
186 \code
187 std::cout << chi_log.PrintEventHistory(tag);
188 \endcode
189 \verbatim
190 1.33333
191 [0] 3.813119000 EVENT_CREATED
192 [0] 3.813120000 SINGLE_OCCURRENCE A
193 [0] 3.813121000 SINGLE_OCCURRENCE B
194 [0] 3.813122000 SINGLE_OCCURRENCE C
195 \endverbatim
196  * */
