14 /**Object for controlling logging.
15 *
16 * ## Part A: Output logs
17 * There are three levels of verbosity in ChiTech: Zero(Default), One and Two.
18 * These can be set on the command line via the switch -v followed by a
19 * space and the number for the verbosity (0,1 or 2).
20 *
21 * \code
22 * ./bin/ChiTech InputFile.lua -v 1
23 * \endcode
24 *
25 * The lua command `chiLogSetVerbosity(int_level)` achieves the same.\n\n
26 *
27 * Printing a log under the auspices of a verbosity level again has
28 * numerous options. Firstly, any log can be a normal log, a warning
29 * or an error. Secondly, a log can be either location 0 or all the locations
30 * in a parallel process environment. The log option enums defined under
31 * LOG_LVL are
32 - LOG_0, Used only for location 0
33 - LOG_0WARNING, Warning only for location 0
34 - LOG_0ERROR, Error only for location 0
35 - LOG_0VERBOSE_0, Default verbosity level
36 - LOG_0VERBOSE_1, Used only if verbosity level equals 1
37 - LOG_0VERBOSE_2, Used only if verbosity level equals 2
38 - LOG_ALL, Verbose level 0 all locations
39 - LOG_ALLWARNING, Warning for any location
40 - LOG_ALLERROR, Error for any location
41 - LOG_ALLVERBOSE_0, Default verbosity level
42 - LOG_ALLVERBOSE_1, Used only if verbosity level equals 1
43 - LOG_ALLVERBOSE_2, Used only if verbosity level equals 2
44
45 * A log can be made by first connecting code with the logger. This is done
46 * by including the log header and then defining an extern reference to the
47 * global object.
48 *
49 * \code
50 * #include <chi::log.h>
51 * extern ChiLog& chi::log;
52 * \endcode
53 *
54 * A log is then written inside a piece of code as follows:
55 *
56 * \code
57 * void PrintSomethingToLog()
58 * {
59 * chi::log.Log() << "This is printed on location 0 only";
60 * chi::log.Log0Warning() << "This is a warning";
61 * chi::log.Log0Error() << "This is an error";
62 * }
63 * \endcode
64 *
65 \verbatim
66 [0] This is printed on location 0 only
67 [0] **WARNING** This is a warning
68 [0] **!**ERROR**!** This is an error
69 \endverbatim
70 *
71 * ## Part B: Repeating events log
72 * Suppose a routine or segment of code gets called multiple times. Now suppose
73 * we want to know how many times this routine was called, how long it ran
74 * (total and on-avg), or we want to know at which timestamp it ran every time.
75 * We do this by using a repeating event. A repeating event is identified
76 * by a tag. Therefore the first step is to obtain a unique id for the event using
77 * ChiLog::GetRepeatingEventTag.
78 *
79 * \code
80 * size_t tag = chi::log.GetRepeatingEventTag(std::string("Sweep Timing"))
81 * \endcode
82 *
83 * Events can now be logged using
84 * ChiLog::LogEvent(size_t ev_tag, ChiLog::EventType ev_type). As an example,
85 * consider we want to count the number of occurrences of an event. We trigger
86 * multiple occurences using
87 *
88 * \code
89 * chi::log.LogEvent(tag,ChiLog::EventType::SINGLE_OCCURRENCE);
90 * \endcode
91 *
92 * At the end of the block for which you wanted to log these events we can now
93 * extract the number of occurrences using
94 *
95 * \code
96 * double number_of_occ = chi::log.ProcessEvent(tag,ChiLog::EventOperation::NUMBER_OF_OCCURRENCES);
97 *\endcode
98 *
99 * Below is a complete example:
100 *
101 \code
102 size_t tag = chi::log.GetRepeatingEventTag(std::string());
103
104 for (int i=0; i<5; ++i)
105 chi::log.LogEvent(tag,ChiLog::EventType::SINGLE_OCCURRENCE);
106
107 chi::log.LogAll()
108 << chi::log.ProcessEvent(tag, ChiLog::EventOperation::NUMBER_OF_OCCURRENCES);
109 \endcode
110 \verbatim
111 6
112 \endverbatim
113
114 * Since chi::log is a global object this functionality is really powerful for
115 * use in modules or class objects where multiple methods can contribute to
116 * the same event track. For example an object can have an event tag as a member
117 * and initialize it at construction then all of the objects methods can
118 * contribute to the event.
119 *
120 * ### Supplying event information
121 * In addition to the ChiLog::EventType the user can also supply a reference to
122 * a ChiLog::EventInfo structure. Be cautious with this though because each
123 * event stored a double and a string which can look like a memory leak if
124 * multiple events are pushed up with event information. Developers can supply
125 * either a double or a string or both to an event info constructor to
126 * instantiate an instance. The event arb_value is by default 0.0 and the event
127 * arb_info is by default an empty string. An example is shown below:
128 *
129 \code
130 size_t tag = chi::log.GetRepeatingEventTag(std::string());
131
132 for (int i=0; i<5; ++i)
133 {
134 auto ev_info = std::make_shared<ChiLog::EventInfo>(i*2.0);
135 chi::log.LogEvent(tag,ChiLog::EventType::SINGLE_OCCURRENCE,ev_info);
136 }
137
138 chi::log.LogAll()
139 << chi::log.ProcessEvent(tag, ChiLog::EventOperation::AVERAGE_VALUE);
140 \endcode
141 \verbatim
142 4.0
143 \endverbatim
144 *
145 * Event information can also be supplied by string values but then the
146 * average value is meaningless unless the average value is also supplied.
147 * For example:
148 *
149 \code
150 size_t tag = chi::log.GetRepeatingEventTag(std::string());
151
152 chi::log.LogEvent(tag,
153 ChiLog::EventType::SINGLE_OCCURRENCE,
154 std::make_shared<ChiLog::EventInfo>(std::string("A"),2.0));
155 chi::log.LogEvent(tag,
156 ChiLog::EventType::SINGLE_OCCURRENCE,
157 std::make_shared<ChiLog::EventInfo>(std::string("B")));
158 chi::log.LogEvent(tag,
159 ChiLog::EventType::SINGLE_OCCURRENCE,
160 std::make_shared<ChiLog::EventInfo>(std::string("C"),2.0));
161
162 chi::log.LogAll()
163 << chi::log.ProcessEvent(tag, ChiLog::EventOperation::AVERAGE_VALUE);
164 \endcode
165 \verbatim
166 1.33333
167 \endverbatim
168 *
169 * To get a string value of the event history developers can use
170 * ChiLog::PrintEventHistory along with the event tag. Just note that it will
171 * automatically be formatted for each location so no need to use chi::log to
172 * print it. Also, each event will be prepended with a program timestamp
173 * in seconds.
174 *
175 \code
176 std::cout << chi::log.PrintEventHistory(tag);
177 \endcode
178 \verbatim
179 1.33333
180 [0] 3.813119000 EVENT_CREATED
181 [0] 3.813120000 SINGLE_OCCURRENCE A
182 [0] 3.813121000 SINGLE_OCCURRENCE B
183 [0] 3.813122000 SINGLE_OCCURRENCE C
184 \endverbatim
185 * */
