138 ###Note on the Eager limit
139 The eager limit is the message size limit before which non-blocking MPI send
140 calls will execute without waiting for a matching receive call. The limit is
141 platform dependent but in general 64 kb. Some systems have 32 kb as a limit
142 and therefore we use that as a default limit in ChiTech. There is a fine
143 interplay between message size and the shear amount of messages that will be
144 sent. In general smaller messages tend to be more efficient, however, when
145 there are too many small messages being sent around the communication system
146 on the given platform will start to suffer. One can gain a small amount of
147 parallel efficiency by lowering this limit, however, there is a point where
148 the parallel efficiency will actually get worse so use with caution.
