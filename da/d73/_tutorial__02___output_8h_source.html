1 /** \page Tutorial02 Tutorial 2: Adding basic output
2  *
3  * Solver modules in ChiTech connect their solution information to a concept
4  * called a <B>Field Function</B>. A field function is considered fully defined
5  * when it is connected to both a grid (mesh) and a spatial discretization.
6 
7  \image html "Physics/FieldFunctionHierarchy.png" "Figure 1 - Hierarchy of field functions" width=600px
8 
9 ## Step 1 - Make a copy of Tutorial01 input
10 In the same folder (or any of your choice) make a copy of the input you used for Tutorial 1.
11 We will be adding some items to this input file.
12 
13 ## Step 2 - Obtain a list of field functions associated with the solver
14 
15  \code
16  fflist,count = chiGetFieldFunctionList(phys1)
17  \endcode
18 
19  The funcation call chiGetFieldFunctionList() provides us with two items. A
20  lua-table and a count of how many items there are in the table. The items
21  in "fflist" are the text names of the field functions. Each solver has its
22  own defaults.
23 
24 ## Step 3 - Create a slice interpolator
25 
26 \code
27 slice1 = chiFFInterpolationCreate(SLICE)
28 chiFFInterpolationSetProperty(slice1,SLICE_POINT,0.0,0.0,1.5)
29 chiFFInterpolationSetProperty(slice1,ADD_FIELDFUNCTION,fflist[1])
30 \endcode
31 
32 The first call creates a interpolator of type SLICE. At the time of writing this
33  tutorial we support LINE, SLICE and VOLUME. The default orientation of a slice
34  interpolator is with the cutting plane's normal pointing in the direction
35  of \f$ \hat{k} \f$ and the reference point at (0,0,0). We can keep the same
36  orientation but move the reference point to the center of the extrusion
37  (i.e. 1.5). This is achieved with a call to chiFFInterpolationSetProperty() with
38  a property index SLICE_POINT. The last line here is to add a field function to
39  this interpolator. We use the same function but this time with a property index
40  ADD_FIELDFUNCTION.
41 
42 ## Step 4 - Initialize and execute
43 For very complex meshes it might be prudent to perform initialization before
44  actually solving the systems, since
45  this established the necessary interpolation parameters and allows one to
46  execute the interpolator multiple times after that with minimal cost.
47 
48 \code
49 chiFFInterpolationInitialize(slice1)
50 chiFFInterpolationExecute(slice1)
51 chiFFInterpolationExportPython(slice1)
52 \endcode
53 
54 The chiFFInterpolationInitialize() and chiFFInterpolationExecute() should be
55  intuitive to understand. The last function call here is chiFFInterpolationExportPython()
56  which is a utility to export a slice to a python file. Inside this python
57  file there are some default visualization commands but the major utility here
58  is that the field function is now represented as python variables so that
59  the user can define custom visualizations.
60 
61 ## Step 5 - Run the python file
62 Being a useful scripting system, the lua console can itself invoke processes.
63  Here the default export name of the python file will be "ZPFFI00.py". Its
64  actually "ZPFFI0", for Plane-Field-Function-Interpolator-0 but the last digit
65  denotes the processor identification in
66  such a way that the user merely needs to execute the 0-index and the rest will
67  be loaded.
68 
69 \code
70 local handle = io.popen("python ZPFFI00.py")
71 \endcode
72

98

104

107 chiVolumeMesherSetProperty(MATID_FROMLOGICAL,vol0,material)
108

111

113 
114

117 chiPhysicsMaterialSetProperty(material,"k",SINGLE_VALUE,1.0)
