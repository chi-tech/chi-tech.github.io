2 
3 Lua is the main scripting system used by ChiTech.
4 
5 \section devman03_sec0 Adding Lua-input
6 
7 ## Step 1: Create a source file
8 First create a `.cc` file in the location where you want the lua-input to
9 reside. Example: `CHI_TECH/lua_test.cc`
10 \n\n
11 If the file is not covered by a `CMakeLists.txt` recursion then follow
12 the instructions for \ref DevManAddingCode "adding a code to the system".
13 
14 ## Step 2: Define a standard lua template with your input function name
15 In our case the test function will be `LuaTest`. We add `chi` to the front of
16 all input language to distinguish it from other lua functions.
17 \n\n
18 Lua-wrapper functions have no real arguments other than the lua-state and
19 always return an integer representing the number of items returned. For more
20 information on how to handle parameters in lua C-api consult the
21 <a href="https://www.lua.org/pil/24.html">lua-documentation</a>.
22 
23 \code
24 #include <ChiLua/chi_lua.h>
25 
26 //###################################################################
27 ///This is a lua test function.
28 int chiLuaTest(lua_State* L)
29 {
30  return 0;
31 }
32 \endcode
33 
34 ## Step 3: Add your code to it
35 In this simple example we will simply take the first argument and print it
36 to `stdout`. We also include argument protection.
37 
38 \code
39 #include <ChiLua/chi_lua.h>
40 #include <iostream>
41 
42 //###################################################################
43 ///This is a lua test function.
44 ///\param argument1 Any Argument of any type.
45 int chiLuaTest(lua_State* L)
46 {
47  //============================== Optional argument protection
48  int num_args = lua_gettop(L);
49  if (num_args<1)
50  LuaPostArgAmountError("chiLuaTest",1,num_args);
51 
52  //============================== Obtain first argument from stack
53  const char* argument_1 = lua_tostring(L,1);
54 
55  //============================== Print to screen
56  std::cout << "LuaTest: " << argument_1 << std::endl;
57 
58  return 0;
59 }
60 \endcode
61 
62 ## Step 4: Add it to the lua-register
63 In the project directory go to the file `CHI_TECH/ChiLua/chi_lua_register.h`
64 and add a `RegisterFunction` call to the register, registering the lua function
65 call to be connected with the lua console.
66 \n\n
67 If you would like for the function call to be listed under a category then
68 proceed the `RegisterFunction` call with a comment starting exactly with
69 `//module:` which will create a module with text matching the text following
70 this piece.
71 \n
72 \n
73 The module tabulation in the manual and the documentation for the parameters
74 will be processed with a script therefore it is important to comment
75 parameters and return values. Some specific notes in the lua-register file
76 
77  - `//module:` provides a categorization of the function on the documenation
78  main page.
79  - `//\ref` will provide a link anchor on the documenation main page.
80 
81 \code
82  .
83  .
84  .
85 RegisterFunction(chiLBSGroupsetSetGroupSubsets)
86 RegisterFunction(chiLBSGroupsetSetIterativeMethod)
87 RegisterFunction(chiLBSGroupsetSetResidualTolerance)
88 RegisterFunction(chiLBSGroupsetSetMaxIterations)
89 RegisterFunction(chiLBSGroupsetSetGMRESRestartIntvl)
90 RegisterFunction(chiLBSGroupsetSetWGDSA)
91 RegisterFunction(chiLBSGroupsetSetTGDSA)
92 
93 //module:Test scripts
94 RegisterFunction(chiLuaTest)
95 
96 
97 \endcode
98 
99 
100 ## Final step: Test it!!
101 
102 Create a script input file and call your function. If it is linked incorrectly
103 the lua scripting system will throw an error like
104 
105 `Test.lua:2: attempt to call a nil value (global 'chiLuaTest')`
